1.3.3. 식별자와 변수 이름

  • 변수 이름은 식별자며 식별자는 대소문자를 구분한다
  • 예약어는 문법의 일부분으로 변수, 함수, 매개변수 이름에 쓸수 없다

1.4 값

  • 불리언, 숫자, 문자열, 배열 등 여러 값 있음
  • 모든 값에는 프로퍼티가 있음
  • 각 프로퍼티에는 키와 값이 있음
  • 점 연산자는 프로퍼티 값을 읽을 수 잇음
  • 점 연산자는 프로퍼티에 값을 할당 할 수 있음
  • 점 연산자는 메서드를 호출 할 때도 사용

1.4.1 원시값 vs 객체

  • 원시 값은 불리언, 숫사, 문자열, null, undefined
  • 다른 값은 모두 객체
  • 각 객체는 고유함
  • 원시값은 값이 같으면 같음

1.4.2 원시값

  1. 불리언
  2. 숫자
  3. 문자열
  4. 값 아닌 값 : undefined, nul
  • 값으로 비교
    : 내용을 비교
  • 항상 불변
    : 프로퍼티를 변경, 추가, 제거 불가

1.4.3 객체

  • 평범한 객체는 객체 리터럴로 만들 수 있음
  • 배열은 배열 리터럴로 만들 수 있음
  • 정규 표현식은 정규 표현식 리터럴로 만들 수있음
  • 참조로 비교
  • 프로퍼티를 자유롭게 추가, 변경, 제거 가능

1.4.4 undefined 와 null

  • undefined 인지 null 인지 체크
  • 명시적 체크
  • 둘 다 false

1.4.5. typeof 와 instanceof 을 통한 값 분류

  • type of : 주로 원시 값 사용
  • typeof value
  • intanceof : value 가 생성자 Constr 에서 만들어진 객체면 true 반환
  • value instanceif Constr

1.5 불리언

  • 원시 불리언 : true, false
  • 2항 논리 연산자 : &&, ||
  • 부정 연산자 : !
  • 비교 연산자
  • 일치 / 동등 연산자 : ===, !==, ==, !=
  • 순서 연산자 : >, >=, =<, <

1.5.1. 참 값은 값, 거짓 같은 값

  • 다음 값은 false -> 나머지는 true
  • undefined, null
  • 불리언 : false
  • 숫자 : -0, NaN
  • 문자열: ‘’

1.5.2 2항 논리 연산자

  • 단축 평가 -> on/off 식
  • 첫번째 피연산만으로 결과를 판단 할 수 있다면 두번째 연산자는 평가 하지 않음
  1. AND &&
  • 첫번째 피연산자가 ‘거짓같은 값’ 이면 첫번째 피연산자 반환
  • 그렇지 않으면 두번째 피연산자 반환
  1. Or ||
    • 첫번째 피연산자가 ‘참 같은 값’ 이면 첫번째 피연산자 반환
    • 그렇지 않으면 두번째 피연산자 반환

1.5.3 동일 연산자

  1. 일반적 비교
  • 동등 : ==
  • 비동등 : !=
  1. 엄격한 비교
  • 일치 : ===
  • 불일치 : !==
